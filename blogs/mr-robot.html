    Findings
    


    1 Sensitive Information Disclosure
    
→ A02:2021 - Cryptographic Failure

Sensitive endpoints and credentials were exposed via publicly accessible files.

Affected:

  • /robots.txt
    • 

  • /license
    • 

Impact:
An attacker can gain initial access without bruteforce.

Mitigation:

  • Remove sensitive data from public files
    • 

  • Restrict access using authentication
    • 

  • Avoid storing credentials in public files
    • 

    

    2 Insecure WordPress Configuration
    
→ A05:2021 - Security Misconfiguration

The WordPress theme editor was enabled, allowing direct modification of PHP files.

Impact:
Leads to Remote Code Execution (RCE).

Mitigation:

  • Disable theme/plugin editor
    • 

    define('DISALLOW_FILE_EDIT', true);
    

  • Restrict admin panel access
    • 

    

    3 Remote Code Execution
    
→ A03:2021 - Injection

The attacker executed arbitrary commands on the server via reverse shell.

Impact:
Full control over application and system operations.

Mitigation:

  • Input validation
    • 

  • Disable dangerous PHP functions
    • 

  • Use Web Application Firewall
    • 

    

    4 Privilege Escalation via SUID Binary
    
→ A05:2021 - Security Misconfiguration

SUID enabled Nmap binary allowed privilege escalation.

Impact:
Complete system takeover.

Mitigation:

  • Remove unnecessary SUID permissions
    • 

    > chmod -s /usr/bin/nmap
    

  • Regular privilege audits
    • 

    

    


    Steps to Reproduce
    


    1. Scanning
    
Tool used: Nmap


    > nmap -sV -sC 192.168.1.41
    


Port 80 is open, which indicates a web server is running on this machine. So I opened the IP address in a browser.



    2. Directory Enumeration
    
Tool used: dirb


    > dirb http://192.168.1.41/ -o dirbResult.txt
> grep "CODE:200" dirbResult.txt
    

Used dirb to enumerate all subdirectories and pages and saved the output to a file. And used grep to extract only the URLs with status code 200 from the dirb output file.


    

    Gathering Key 1 of 3
    
Sent requests to each endpoint and examined the responses using Burp Suite.


/robots.txt exposes the path to access key 1 of 3.


Used the exposed path from robots.txt to retrieve key 1 of 3.

Key 1 of 3:


    073403c8a58a1f80d943455fb30724b9
    


    


    Gathering Key 2 of 3
    
Found an encoded credential pair in /license and decoded it using Burp Decoder.


Credential: 


    elliot:ER28-0652
    

Navigated to /wp-login.php.


Used previously captured credentials on the login page. And the login succeeded.


The file at http://192.168.1.35/wp-admin/theme-editor.php?file=404.php&theme=twentyfifteen was editable. Referenced revshells.com to select a suitable reverse shell payload and replaced the existing code with the reverse shell.


Started a netcat listener to catch the reverse shell.


The payload was placed in 404.php, which executes on a 404 (wrong path); an arbitrary non-existent path was requested to trigger it.


Connection established and reverse shell obtained.


Logged in as daemon; current working directory: /.
Navigated to /home/robot.


Attempted to read key-2-of-3.txt from /home/robot but user daemon doesn't have the permission to access the file.


Opened password.raw-md5 (accessible to daemon). It contained a username and an MD5-hashed password.


Used CrackStation to recover the plaintext password from the MD5 hash.


Credential:


    robot:abcdefghijklmnopqrstuvwxyz
    

Logged in as robot using the recovered credentials.


key-2-of-3.txt is now accessible and was opened.

Key 2 of 3:


    822c73956184f694993bede3eb39f959
    


    


    Gathering Key 3 of 3
    
There is nothing else in user account. So escalation to root is required.

Unable to perform required actions as root, so searched for files runnable with root privileges to leverage for escalation.


    > find / -perm -u=s -type f 2>/dev/null
    


The list included Nmap, so gtfobins.org was consulted to learn how to perform privilege escalation using Nmap.

Obtained admin access.


Navigated to /root, opened key-3-of-3.txt, and retrieved the final key.

Key 3 of 3:


    04787ddef27c3dee1ee161b21670b4e4
